# 拾级 StepUp - 需求文档 v0.1

**日期**: 2026-04-03  
**版本**: v0.1

---

## 项目概述

本项目旨在构建一个基于人工智能的个性化学习辅助平台（支持 Web 端与微信小程序），通过深度分析学生的考试试卷，精准定位知识薄弱点，并生成定制化的改进计划与针对性练习。系统致力于实现从"题海战术"到"精准学习"的转变，帮助高中生高效提升学业成绩。

---

## 用户角色

| 角色 | 说明 |
|------|------|
| **学生** | 系统的主要使用者，进行试卷上传、查看报告、完成练习 |
| **家长，可选）** | 查看孩子的学习报告与进步曲线 |
| **管理员** | 负责题库维护、PI 模型参数配置及内容审核 |

## 系统模块

系统分为三个部分：

### 1. 后台管理系统

这是整个平台的"大脑"和"控制台"，主要供管理员和教研人员使用。它基于 Web 浏览器运行，核心职能是进行全局的数据维护与系统配置。技术上，它通过权限控制接口，负责管理用户账户体系、录ᅥ和分类海量题库资源、配置 AI 模型的参数，以及查看平台整体的运行数据报表。它是数据录入和规则制定的入口，确保前台应用有充足的"弹药"和稳定的运行规则。

### 2. 浏览器应用

这是面向学生用户的"沉浸式学习工作站"，同样基于 Web 浏览器运行，能适配 PC 端大屏、手机和平板小屏显示。它侧重于深度交互和可视化展示，利用 Go 后端的高效渲染和 HTMX 的动态刷新能力，为学生提供复杂的图表分析（如学情雷达图）、长文本题目的阅读与作答、以及系统化的学习路径规划。相比移动端，它更适合进行长时间的专注学习、试卷的高清查看以及复杂的错题整理操作。

### 3. 微信小程序

这是面向学生用户的"移动便携终端"，依托微信生态运行。它侧重于轻量化、即时性和便捷性，利用小程序的 API 态刷新，核心解决"随时随地"的学习需求。技术上，它主要负责调用手机摄像头进行试卷拍照上传、接收 AI 分析完成后的即时服务通知、以及利用碎片化时间进行快速刷题和查看简易报告。它是 PC 端学习场景的延伸，强调快速记录和即时反馈。

## 系统技术需求

| 组件 | 技术栈 |
|------|--------|
| 架构工具 | Go + Gin + GORM + templ |
| 数据库 | MySQL |

### 数据库设计约束

- 关键业务数据只能软删除，不允许物理删除
- 涉及启停状态的字段统一使用 `status`，取值规范为：active = 1，inactive = 0
- 每个记录都必须有 `created_at` 和 `created_by`
- 可修改的记录必须有 `updated_at` 和 `updated_by`

## 后台管理功能需求

### 1. 学生 CRTD

- 学生记录创建以后，可以登录浏览器应用
- 使用手机号码或者邮箱登录系统
- 可以查看学生上传的 PDF 或者图片、AI 的试卷分析结果、AI 的改进计划以及在系统做过的题目、练习

### 2. 浏览器应用 AI 模型设置

- 这里配置浏览器应用 AI 模型的数据，包括 AI URL、app_key、app_secret 等参数
- 浏览器应用根据这个 AI 设置来使用 AI 大模型
- 可以配置多个模型，但只有一个处于激活状态，可以简单切换选择模型

### 3. 系统科目设置

- 系统目前支持的科目，目前暂时只设置物理和语文两科，后面陆续增加

### 4. 学生阶段设置

- 系统目前支持的阶段，目前只有高中。将来可能有初中和小学

### 5. 系统 Prompt 设置

- 配置所有地方使用到 AI 大模型的 Prompt 都在这里设置
- 使用的地方通过 key 来数据库读取
- 有 key 和简单说明

### 6. 系统 Audit Log 查询

- Audit log 按照系统三部分，三个系统都需要做 audit log
- 所有用户操作都需要有记录，包括用户登录、修改密码
- 如果是修改或者删除记录，要有 snapshot 字段记录操作前的数据记录，用 JSON 格式保存起来
- 敏感数据不需要保存

---

## 浏览器应用功能需求

### 1. 登录页面

- 用户可以输入自己的手机号码或者邮箱来登录，如果用户记录不存在系统
- 系统会通过手机号码或者邮箱发送验证码来验证，验证通过以后需要设置密码，系统会在后台创建学生记录
- 用户注册以后，通过密码来登录系统

### 2. 主页面

- 现在系统里面的学习科目有：物理、语文（可在系统后台配置）
- 用户可以选择科目，上传 PDF 或者图片来做试卷分析
- 用户提交文件以后，系统后台设置好 AI 模型的：
  - **system prompt**：说清楚系统的角色和职责范围，满足目前的系统需求的一个说明
  - **user prompt**：说清楚学生的情况，所在阶段，需要评估他的试卷，根据试卷评估结果和试卷反应出来的问题做一个学习改进计划
- 把 AI 的返回结果暂时给用户看，同时这些所有数据记录都需要保存到数据库
- 可以在后台和浏览器应用都可以看到
